Diagrama de Entidades e Relacionamentos (ERD)

Mapeamento lógico de como os usuários, empresas, vagas, candidaturas e perfis se relacionam no sistema de banco de dados.

Tabela de Usuários

Contém o registro geral e credenciais de login. O atributo role dita se o usuário é candidato, recrutador, admin ou master.

Perfis Específicos

Utiliza uma estrutura de especialização (1:1). candidate_profiles armazena dados de currículo, e recruiter_profiles gerencia o acesso às empresas.

Muitos-para-Muitos

Recrutadores podem se associar a múltiplas empresas por meio da tabela pivô companie_recruiter_profile, que contém um campo approved de aprovação.

Fluxo de Cadastro de Usuário (Padrão Factory Method)

Fluxo demonstrando a criação dinâmica de perfis. O sistema utiliza uma fábrica (UserProfileFactory) para instanciar o criador correto de acordo com a escolha do usuário.

Nova Empresa

Quando o recrutador seleciona "Nova Empresa", o sistema cria simultaneamente a empresa e define o usuário como Admin e dono dela, ativando privilégios de gerenciamento.

Empresa Existente

Se selecionar "Empresa Existente", o recrutador é vinculado a ela como pendente (approved = false). Ele precisará que o Administrador ou o Master o aprove para acessar os recursos.

Ciclo de Candidatura & Eventos (Observer Pattern)

A jornada de uma candidatura a uma vaga. Quando a candidatura é aprovada, o sistema dispara um evento para fechar a vaga automaticamente e de forma desacoplada.

sequenceDiagram autonumber actor Candidato as Candidato (Student) actor Recrutador as Recrutador / Admin participant AppService as ApplicationService participant Event as ApplicationApproved (Event) participant Listener as CloseJobListener (Observer) participant Job as Vaga (Job Model) Candidato->>AppService: Envia candidatura à Vaga (cria registro 'pendente') Recrutador->>AppService: Altera status para 'aprovado' activate AppService AppService->>Event: Dispara Evento (ApplicationApproved) deactivate AppService activate Event Event->>Listener: Notifica Listener cadastrado deactivate Event activate Listener Listener->>Job: Executa update(['status' => 'fechada']) deactivate Listener Note right of Job: A vaga muda para 'fechada' e encerra novos applies

1. Inscrição

O Candidato preenche a carta de apresentação. Ele pode se candidatar apenas se a vaga estiver ativa e se ele tiver o perfil do tipo candidato.

2. Análise e Acolhimento

Recrutadores da empresa vinculada analisam as candidaturas de forma privada no painel e alteram o status para 'em_analise', 'reprovado' ou 'aprovado'.

3. Desacoplamento

Graças ao padrão Observer, novos comportamentos pós-aprovação (como envio de e-mails ou integração com WhatsApp) podem ser adicionados criando novos Listeners sem alterar as funções principais.

Controle Multitenant

Recrutadores só enxergam e gerenciam as vagas das empresas às quais estão vinculados e aprovados. O sistema filtra as visualizações baseando-se no ID do recrutador logado.

Ações do Master

O Super Admin (Master) possui super-privilégios, podendo pular a aprovação padrão e vincular recrutadores a qualquer empresa diretamente pelo painel.
